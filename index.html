01

Complete · Live

Feed The City Event Map

A searchable national map so volunteers find the nearest meal-packing event and jump straight to official Tango registration — replacing a wall of static city links.

  • Search by city, ZIP, or address, ranked by distance
  • Google Sheets–backed, maintained by non-technical leaders
  • Weekly sentinel watches the source site for changes
  • Live on Cloudflare Pages — ~48 events nationwide

FTC Distribution Tool

A planning layer that matches an event's meal output to partner organizations that can actually receive it — confirming capacity ahead of time and finding backups before food goes to waste.

  • Event ↔ partner matching with capacity limits
  • Pre-event confirmation workflow for leaders
  • Backup discovery for overflow — planned
  • Public-safe by design — sample data only

03

Prototype · active dev

Volunteer Experience Platform

Turns a volunteer's sandwich goal into an exact ingredient and shopping plan, then hands off to city registration — so "what to bring" stops being guesswork.

  • Goal → ingredient quantities → shopping list
  • Per-city sign-up via URL parameter
  • React + Vite, deployed on Vercel
  • Leader check-in & reporting — planned

How I work

AI lets me build at speed. The judgment — what to build, what's safe to ship, and how it's handed off — stays mine.

I use Claude and Claude Code throughout: prototyping, debugging, refactoring, and documentation. Product direction, nonprofit context, data-privacy decisions, and handoff planning are owned by me — so each tool survives beyond the person who built it.
