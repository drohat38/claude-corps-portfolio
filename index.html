Feed The City Event Map

The Feed The City Event Map helps volunteers find nearby Feed The City meal-packing events and click through to official Tango Charities registration pages. I built it after realizing that the national Feed The City page listed city links in static columns, forcing volunteers to manually click through each page to find time, location, cadence, and registration details.

Problem

Volunteers needed a faster way to discover nearby events without clicking through dozens of city pages.

What I built

  • Searchable public map (city, ZIP, or address search)
  • Filters by state, Saturday cycle, and event time
  • Google Maps pins colored by Saturday cycle, ranked by distance
  • Google Sheets-backed event database (published CSV)
  • Apps Script admin workflow for nontechnical maintenance
  • Weekly source-site change sentinel that watches the Tango page
  • Cloudflare Pages deployment (auto-deploys on push)
  • Sign-up links straight to official Tango city pages

What I learned

This project taught me that nonprofit technology has to work for multiple users at once: volunteers on phones, leaders managing events, and staff maintaining data after the original builder steps away.

FTC Distribution Tool

The FTC Distribution Tool is an internal-operations planning concept for matching Feed The City events with partner organizations that can receive meals. It came from a real problem I experienced in Denton: when an event produces more meals than one partner can accept, leaders need a structured way to identify backup partners, confirm capacity, and prevent food from going to waste.

Problem

Volunteer enthusiasm can create operational pressure if meal volume exceeds partner capacity. Leaders need a system for capacity, backups, and accountability.

What I am building

  • Partner and event records, linked by a stable event ID
  • Event-to-partner assignment
  • Capacity confirmation workflow before events
  • Leader reminders on a recurring cycle
  • Planned backup partner discovery
  • Planned food-safety / refrigeration capacity fields
  • Planned clear public/private data boundary

Public-safe note: this public repository should contain only prototype, architecture, or sanitized example data. Real partner contact information, meal capacity details, agreements, and operational records should remain private.

Feed The City Volunteer Experience Platform

The Volunteer Experience Platform is my active work to improve how volunteers and leaders interact with Feed The City. The current volunteer-planner prototype helps volunteers turn a sandwich goal into an ingredient plan and shopping-list style output, then connect that planning experience to registration.

Problem

Static "what to bring" instructions are hard for volunteers to translate into an actual shopping plan.

What I built / am building

  • Sandwich-goal input
  • Ingredient quantity calculator
  • Receipt / shopping-list style output
  • City sign-up / registration flow
  • Wix iframe integration (per-city via URL parameter)
  • Planned leader check-in & attendance
  • Planned meal reporting & Eventbrite gap-fixing
  • Planned service-hour verification

Built with React + Vite and deployed on Vercel as a prototype. The broader volunteer/leader system is active development, not a finished product.
